# Feedback Note

This is a template. A template is a fill-in-the-blank form, like a birthday card that already has "Happy Birthday" printed and just leaves a space for the name. You do not start from a blank page. You only fill in the blanks.

**How to use it:**

1. Copy everything below the line into a new note.
2. Fill in each blank with your own words.
3. Write **one note per issue or idea.** If you found three things, make three separate notes. One thing per note keeps each report clear.
4. If a field does not fit your issue, just leave it blank. That is fine.

Not sure how to word something? MILO can help you turn rough notes into a clean report. There is an example prompt in [`lessons/07-giving-good-feedback.md`](../lessons/07-giving-good-feedback.md).

**Title:**
_(A short name for this note, like "Button was hard to find".)_

**What I was doing:**
_(The steps you took, in order. Example: 1) Opened the app. 2) Tapped Start.)_

**What I expected:**
_(What you thought would happen.)_

**What actually happened:**
_(What really happened instead.)_

**Where (screen or page):**
_(The name of the screen or page you were on.)_

**Device:**
_(Phone, tablet, or computer. Add the kind if you know it, like "iPhone".)_

**Screenshot:**
_(Drag in a picture of your screen, or write "none". A screenshot is just a photo of what your screen showed.)_

**How much it bugged me:**
_(Pick one: small / medium / big.)_

**Why it matters (optional):**
_(Who this could affect, and how much. Example: "A new user might think the app froze.")_

**Idea to fix (optional):**
_(Any idea you have. It is totally fine to leave this empty.)_
