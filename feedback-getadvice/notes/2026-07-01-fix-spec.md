# Fix Spec — App Freeze on Normal Questions

## Problem

When a user sends a normal question in GetAdvice's chat screen, the app shows no loading indicator while it waits for a reply. During that wait the screen shows no visible change and stops responding to taps, so it looks and feels frozen — especially for questions like "what model are you?" or "when was your last update?".

## Proposed change

As soon as the user sends a message, show a visible "thinking" indicator in the chat so the user knows the app is working, not stuck. Keep the message input usable the whole time, so the user never loses the ability to interact with the screen. If a reply takes longer than a few seconds, keep the loading indicator visible instead of leaving a silent gap. If a response fails or never arrives, show a short error message instead of leaving the screen stuck indefinitely.

## How we verify

1. Open GetAdvice's chat screen.
2. Send a normal question (e.g. "What model are you?").
3. Send a second question (e.g. "When was your last update?").

**New expected result:** A loading indicator appears immediately after each send, the input stays usable the whole time, and either a reply or a clear error message shows up within a reasonable wait — the screen never looks stuck with no feedback.

## Out of scope

This does not address whether the answers themselves are accurate (that is tracked separately in the "incorrect info" feedback note). It does not change how long the AI actually takes to respond — only how that wait is communicated to the user. Rewriting the underlying chat or network code is out of scope; this only proposes a loading-state UI behavior.
